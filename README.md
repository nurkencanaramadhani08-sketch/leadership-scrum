## Deskripsi Proyek

Proyek ini merupakan simulasi penerapan Scrum pada pengembangan aplikasi Mobile Kantin Pintar Kampus. Tim bertugas menyelesaikan proyek yang mengalami keterlambatan, konflik komunikasi, dan kurangnya dokumentasi dengan waktu pengerjaan dua minggu.

---

## Team Manifesto

- Menghargai seluruh anggota tim.
- Tidak saling menyalahkan.
- Daily Scrum setiap pukul 09.00 WIB.
- Semua perubahan API wajib didokumentasikan.
- Seluruh kode harus melalui Code Review.
- Fokus pada penyelesaian masalah.
- Menjaga komunikasi yang profesional.

---

## SOP Code Review

### Langkah Review

1. Membuat Pull Request.
2. Reviewer memeriksa kualitas kode.
3. Memberikan komentar yang sopan dan konstruktif.
4. Developer melakukan perbaikan.
5. Merge setelah disetujui.

### Contoh Komentar

✔ "Bagian ini dapat disederhanakan menggunakan fungsi terpisah."

✔ "Validasi input dapat ditambahkan untuk meningkatkan keamanan."
